# CampusHelpATW 联调验收清单（MVP）

## 1. 数据准备
1. 执行建表：`mysql_schema_v0.1.sql`。
2. 执行种子：`mysql_seed_v0.1.sql`。
3. 执行守卫：`mysql_guards_v0.1.sql`。
4. 执行联调数据：`mysql_mock_data_v0.1.sql`。
5. 执行检查：`mysql_sanity_checks_v0.1.sql`。

## 3. 任务状态验收
1. 应存在 10 条 `联调任务-*`。
2. 状态分布应为：
`OPEN=3 / ACCEPTED=2 / IN_PROGRESS=2 / PENDING_CONFIRM=1 / DONE=1 / DISPUTED=1`
3. 非 `OPEN` 任务应存在状态日志。

## 4. 聊天与评价验收
1. 联调任务聊天消息应至少 8 条。
2. `联调任务-09-已完成` 应存在双方评价共 2 条。
3. 任务 09 对应积分流水应至少 3 条。

## 5. 举报与后台验收
1. `联调任务-10-申诉中` 应至少有 1 条任务举报。
2. 应存在对应管理员处理日志。

## 6. 接口联调建议顺序
1. 登录并获取 token。
2. 拉取任务列表，验证状态分布。
3. 拉取任务详情 + 状态日志。
4. 拉取任务聊天消息。
5. 拉取评价列表与积分流水。
6. 拉取举报列表与管理端处理记录。

## 7. 常见问题
1. 重复导入联调数据无变化：
脚本有一次性标记 `seed.mock_data_v0_1_applied`，需先删除该配置再重跑。

2. 状态流转报错：
触发器限制非法跳转，请按状态机顺序调用接口或过程。

3. 积分余额与流水不一致：
请统一走 `sp_add_points`，不要直接改余额字段。

4. 需要重置联调环境：
先执行 `mysql_mock_data_cleanup_v0.1.sql`，再执行 `mysql_mock_data_v0.1.sql`。
