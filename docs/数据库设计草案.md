# CampusHelpATW 数据库设计草案（MVP）

## 1. 设计目标
1. 支撑任务交易闭环与社交基础能力。
2. 保证状态一致性与数据可追溯性。
3. 优先简单可实现，保留后续扩展点。

## 2. 技术建议
1. 数据库：MySQL 8.x 或 PostgreSQL 15+（二选一即可）。
2. 字符集：UTF-8。
3. 时间字段统一使用 `created_at`、`updated_at`、`deleted_at`（软删除按需）。

## 3. 核心枚举（建议）

### 3.1 任务状态 `task_status`
1. `OPEN`：待接单
2. `ACCEPTED`：已接单
3. `IN_PROGRESS`：进行中
4. `PENDING_CONFIRM`：待确认完成
5. `DONE`：已完成
6. `CANCELED`：已取消
7. `DISPUTED`：申诉中

### 3.2 举报对象类型 `report_target_type`
1. `TASK`
2. `POST`
3. `CHAT_MESSAGE`
4. `USER`

### 3.3 举报状态 `report_status`
1. `PENDING`
2. `PROCESSING`
3. `RESOLVED`
4. `REJECTED`

## 4. 数据表设计

### 4.1 用户表 `users`
用途：保存账号基础信息和信誉聚合数据。

关键字段：
1. `id` bigint pk
2. `campus_email` varchar(128) unique not null
3. `password_hash` varchar(255) not null
4. `nickname` varchar(64) not null
5. `avatar_url` varchar(512) null
6. `school_name` varchar(128) null
7. `college_name` varchar(128) null
8. `reputation_score` int default 0
9. `help_points_balance` int default 0
10. `honor_points_balance` int default 0
11. `status` varchar(16) default 'ACTIVE'
12. `created_at` datetime not null
13. `updated_at` datetime not null

索引建议：
1. unique(`campus_email`)
2. index(`status`, `created_at`)

### 4.2 邮箱验证码表 `email_verification_codes`
用途：注册/登录验证码校验。

关键字段：
1. `id` bigint pk
2. `campus_email` varchar(128) not null
3. `code_hash` varchar(255) not null
4. `expire_at` datetime not null
5. `used_at` datetime null
6. `created_at` datetime not null

索引建议：
1. index(`campus_email`, `expire_at`)

### 4.3 任务表 `tasks`
用途：任务主信息与状态。

关键字段：
1. `id` bigint pk
2. `publisher_id` bigint not null fk -> `users.id`
3. `acceptor_id` bigint null fk -> `users.id`
4. `title` varchar(128) not null
5. `description` text not null
6. `category` varchar(32) not null
7. `location_text` varchar(255) null
8. `reward_amount` decimal(10,2) default 0
9. `reward_type` varchar(16) default 'NONE'  // NONE|CASH|POINTS
10. `deadline_at` datetime not null
11. `status` varchar(32) not null default 'OPEN'
12. `accepted_at` datetime null
13. `completed_at` datetime null
14. `canceled_at` datetime null
15. `created_at` datetime not null
16. `updated_at` datetime not null

约束建议：
1. `publisher_id <> acceptor_id`
2. `deadline_at > created_at`

索引建议：
1. index(`status`, `category`, `created_at`)
2. index(`publisher_id`, `created_at`)
3. index(`acceptor_id`, `created_at`)

### 4.4 任务状态流转日志 `task_status_logs`
用途：记录每次状态变化，便于审计与排查。

关键字段：
1. `id` bigint pk
2. `task_id` bigint not null fk -> `tasks.id`
3. `from_status` varchar(32) null
4. `to_status` varchar(32) not null
5. `operator_user_id` bigint not null fk -> `users.id`
6. `reason` varchar(255) null
7. `created_at` datetime not null

索引建议：
1. index(`task_id`, `created_at`)

### 4.5 任务评价表 `task_reviews`
用途：任务完成后的双向评价。

关键字段：
1. `id` bigint pk
2. `task_id` bigint not null fk -> `tasks.id`
3. `reviewer_id` bigint not null fk -> `users.id`
4. `reviewee_id` bigint not null fk -> `users.id`
5. `rating` tinyint not null  // 1~5
6. `content` varchar(500) null
7. `created_at` datetime not null

约束建议：
1. unique(`task_id`, `reviewer_id`)  // 每人每任务一条评价
2. `reviewer_id <> reviewee_id`

### 4.6 积分流水表 `point_ledger`
用途：帮助点/荣誉点变更明细。

关键字段：
1. `id` bigint pk
2. `user_id` bigint not null fk -> `users.id`
3. `point_type` varchar(16) not null  // HELP|HONOR
4. `change_amount` int not null
5. `balance_after` int not null
6. `biz_type` varchar(32) not null  // TASK_COMPLETE|TASK_PUBLISH|PENALTY...
7. `biz_id` bigint null              // 关联任务或举报等对象
8. `remark` varchar(255) null
9. `created_at` datetime not null

索引建议：
1. index(`user_id`, `point_type`, `created_at`)
2. index(`biz_type`, `biz_id`)

### 4.7 帖子表 `posts`
用途：论坛内容主表。

关键字段：
1. `id` bigint pk
2. `author_id` bigint not null fk -> `users.id`
3. `title` varchar(128) not null
4. `content` text not null
5. `like_count` int default 0
6. `comment_count` int default 0
7. `status` varchar(16) default 'NORMAL'
8. `created_at` datetime not null
9. `updated_at` datetime not null

索引建议：
1. index(`author_id`, `created_at`)
2. index(`status`, `created_at`)

### 4.8 帖子评论表 `post_comments`
用途：帖子评论。

关键字段：
1. `id` bigint pk
2. `post_id` bigint not null fk -> `posts.id`
3. `author_id` bigint not null fk -> `users.id`
4. `content` varchar(1000) not null
5. `created_at` datetime not null

索引建议：
1. index(`post_id`, `created_at`)

### 4.9 点赞关系表 `post_likes`
用途：帖子点赞去重。

关键字段：
1. `id` bigint pk
2. `post_id` bigint not null fk -> `posts.id`
3. `user_id` bigint not null fk -> `users.id`
4. `created_at` datetime not null

约束建议：
1. unique(`post_id`, `user_id`)

### 4.11 任务聊天会话 `task_chats`
用途：任务级别聊天会话。

关键字段：
1. `id` bigint pk
2. `task_id` bigint not null unique fk -> `tasks.id`
3. `created_at` datetime not null

### 4.12 聊天消息表 `chat_messages`
用途：任务聊天消息。

关键字段：
1. `id` bigint pk
2. `chat_id` bigint not null fk -> `task_chats.id`
3. `sender_id` bigint not null fk -> `users.id`
4. `message_type` varchar(16) default 'TEXT'
5. `content` text not null
6. `created_at` datetime not null

索引建议：
1. index(`chat_id`, `created_at`)

### 4.13 举报表 `reports`
用途：统一举报入口。

关键字段：
1. `id` bigint pk
2. `reporter_id` bigint not null fk -> `users.id`
3. `target_type` varchar(32) not null
4. `target_id` bigint not null
5. `reason_code` varchar(32) not null
6. `reason_text` varchar(500) null
7. `status` varchar(16) default 'PENDING'
8. `handler_admin_id` bigint null
9. `handle_result` varchar(500) null
10. `handled_at` datetime null
11. `created_at` datetime not null

索引建议：
1. index(`target_type`, `target_id`)
2. index(`status`, `created_at`)

### 4.14 管理员操作日志 `admin_operation_logs`
用途：记录审核与仲裁操作。

关键字段：
1. `id` bigint pk
2. `admin_user_id` bigint not null fk -> `users.id`
3. `operation_type` varchar(32) not null
4. `target_type` varchar(32) not null
5. `target_id` bigint not null
6. `detail` text null
7. `created_at` datetime not null

### 4.15 任务分类表 `task_categories`
用途：维护任务分类配置，支持后台可配置和前端动态展示。

关键字段：
1. `id` bigint pk
2. `code` varchar(32) not null unique
3. `name` varchar(64) not null
4. `sort_order` int default 0
5. `is_active` tinyint default 1
6. `created_at` datetime not null
7. `updated_at` datetime not null

### 4.16 系统配置表 `system_configs`
用途：保存积分规则、风控阈值等可配置项。

关键字段：
1. `id` bigint pk
2. `config_key` varchar(128) not null unique
3. `config_value` json not null
4. `description` varchar(255) null
5. `created_at` datetime not null
6. `updated_at` datetime not null

### 4.17 管理员角色表 `admin_roles`
用途：定义后台角色。

关键字段：
1. `id` bigint pk
2. `role_code` varchar(32) not null unique
3. `role_name` varchar(64) not null
4. `created_at` datetime not null
5. `updated_at` datetime not null

### 4.18 管理员角色绑定表 `admin_user_roles`
用途：建立管理员用户与角色的多对多关系。

关键字段：
1. `id` bigint pk
2. `user_id` bigint not null fk -> `users.id`
3. `role_id` bigint not null fk -> `admin_roles.id`
4. `created_at` datetime not null
5. `updated_at` datetime not null

约束建议：
1. unique(`user_id`, `role_id`)

## 5. 关键一致性策略

1. 任务接单和状态更新必须使用事务。
2. 积分变更必须同时更新 `users` 余额与 `point_ledger` 流水，且在同一事务内。
3. 评价写入前校验任务状态为 `DONE`。
4. 举报处理与处罚动作要记录在 `admin_operation_logs`。

## 6. 后续扩展预留

1. 如需支持多人抢单，可新增 `task_bids` 表并调整接单逻辑。
2. 如需现金担保，可新增 `wallet_accounts`、`wallet_transactions`。
3. 如需推荐系统，可新增 `task_exposure_logs`、`recommendation_feedbacks`。

## 7. 建表优先级

1. 第一批（必须）：`users`、`tasks`、`task_status_logs`、`task_reviews`、`point_ledger`。
2. 第二批（必要）：`task_chats`、`chat_messages`、`reports`、`admin_operation_logs`。
3. 第三批（社区）：`posts`、`post_comments`、`post_likes`。
4. 第四批（配置与后台）：`task_categories`、`system_configs`、`admin_roles`、`admin_user_roles`。
